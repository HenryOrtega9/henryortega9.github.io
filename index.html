Where this started

My entire life runs through an Obsidian vault: CFA study plans, MBA coursework, training logs, financial models, equity research. It works as a second brain, and Claude Code is the agent that operates it: retrieving context across hundreds of notes, auditing the vault's structure, drafting and filing new material, and remembering things between sessions.

Once an AI agent effectively lived in my notes, every project here followed from the same question: where else do I want to reach it? Inside Obsidian as a first-class chat panel. From my wrist mid-run. As a glance across the desk while it works. In the Office files sitting between the notes. Each tool below is one answer, and they all share the same Mac, the same vault, and the same agent.

AskClaude

ask-claude-watch →
SwiftSwiftUIwatchOS WidgetKitbackground URLSession

A standalone Apple Watch app for talking to Claude Code from the wrist. Dictate a question, get an answer with full conversation context, switch models and reasoning effort from a Settings picker.

  • Done notifications. Lowering the wrist mid-turn arms a background URLSession long-poll; when the turn finishes, watchOS wakes the app and a local notification delivers the reply. No APNs, no server push.
  • Sessions tab. Lists every live Claude session on the Mac and can send input into tmux-hosted ones.
  • Complications. Activity-style dual rings render plan-limit usage on the watch face, refreshed inside the WidgetKit budget.
  • Hardened by a multi-agent review: 21 agents swept the codebase, 10 confirmed bugs fixed, 4 false positives rejected by adversarial verification.

claude-cli-chat

claude-cli-chat →
TypeScriptObsidian API stream-jsonsubprocess orchestration

An Obsidian plugin that turns the Claude Code CLI into a native chat UI inside my knowledge vault. Each tab spawns its own CLI subprocess and renders the line-delimited JSON event stream as bubbles, tool rows, and file diffs.

  • Built against the wire protocol. The plugin is a programmatic client over claude --output-format stream-json, with a typed parser and a documented list of protocol gotchas found the hard way.
  • Remote Control mode drives a full interactive session through a Python PTY proxy.
  • Per-vault MCP management, file pins for context, a model and effort picker, and auto-titled tabs.

watch-bridge

claude-cli-chat/scripts →
Python (stdlib only)PTY launchdTailscaleOAuth

The Mac-side daemon behind the watch app. It holds one interactive Claude session on a pseudo-terminal, parses the TUI, and exposes the session as a clean bearer-authed HTTP API bound only to the Tailscale interface.

  • Drives a terminal UI programmatically: auto-accepts trust and permission dialogs, replays sticky model choices into respawned sessions, and guards against swallowed keystrokes with timed nudges.
  • Turn lifecycle engine with a reply budget: long turns return 202 partials the client can poll, and a long-poll endpoint backs the watch's background notifications.
  • Session directory discovers every live Claude process on the machine and maps each one to its transcript.

TC001 status display

claude-cli-chat/scripts →
PythonAWTRIX 3 32×8 RGB matrixpixel animation

A physical status light for Claude Code: an Ulanzi TC001 LED matrix on my desk mirrors the agent's live state. A glance tells me whether it is thinking, running tools, waiting for approval, or done, without switching windows.

  • Hand-drawn pixel animations per state, pushed at 300 ms intervals by a launchd daemon polling the plugin's state emitter.
  • Frames are run-length compressed into rectangle draw calls, because the firmware rejects payloads past roughly 100 draw items.
  • The animation shown here is the device's actual frame data, exported from the animator's renderers and replayed at its real 300 ms push cadence: thinking verbs, the ready scanner, the permission alert, the DONE! celebration hop, and the idle crab walk with its sleep pause.

Microsoft Office Viewer

obsidian-msoffice-viewer →
TypeScriptPDF.js ExcelJSLibreOffice headless

An Obsidian plugin that renders Word, PowerPoint, and Excel files read-only inside the vault, so coursework and financial models stay browsable next to the notes that reference them.

  • Three rendering pipelines: docx-preview for Word, a headless LibreOffice-to-PDF conversion drawn with PDF.js for PowerPoint, and an ExcelJS grid with frozen panes and merged cells for Excel.
  • Claude-driven file creation: a modal shells out to the Claude Code CLI to generate new Office files straight into the vault.
  • Native-app handoff for the cases where read-only is not enough.
